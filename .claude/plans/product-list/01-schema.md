## Table: `product_listings`

```php
Schema::create('product_listings', function (Blueprint $table) {
    $table->id();
    $table->foreignId('product_id')
        ->constrained('products')
        ->cascadeOnDelete();   // listings die with their product
    $table->string('title', 200);
    $table->string('slug', 220)->unique();     // managed by spatie/laravel-sluggable
    $table->string('visibility', 20)->default('draft'); // public, private, draft
    $table->boolean('is_active')->default(true);
    $table->timestamps();
    $table->softDeletes();

    $table->index('product_id');
    $table->index('visibility');
    $table->index('is_active');
    $table->index('deleted_at');
});
```

## Key Design Decisions

### No price columns on this table
Prices (`regular_price`, `sale_price`) live on the parent Product. All listings for a product share the same prices.

### spatie/laravel-sluggable for slug generation
`HasSlug` trait on the model. Generates slug from a custom `generateSlugFrom` method that prepends `product.sku`.
Handles uniqueness automatically (appends `-2`, `-3`, etc. on collision).
Slug field is NOT in `$fillable` — sluggable writes it directly.

### Mutable slug + redirect history
Slug regenerates on title change. Old slugs are preserved for 301 redirects.
See `product-slug/01-schema.md` and `product-slug/00-overview.md` for full design rationale.

### visibility vs is_active
- `visibility = public` + `is_active = true` → shown on storefront
- `visibility = private` → admin-only preview
- `visibility = draft` → not shown anywhere
- `is_active = false` → instantly hides regardless of visibility
