# Purchase Order Module — Overview

## Purpose
Manage purchase orders placed with suppliers. A PO records the intent to purchase inventory —
what products, how many, at what price. When physical units arrive they are received against
the PO, triggering the pipeline process.

## PO Number
Auto-generated by system on creation: `PO-2026-0001`, `PO-2026-0002`, etc.
Sequential within the year, displayed with year prefix, never resets, never editable.

Format: `PO-YYYY-XXXX` where YYYY = year of creation, XXXX = zero-padded 4-digit sequence within that year.

## PO Status Lifecycle

```
draft → open → partial → closed
         └───────────────→ cancelled
```

| Status | Meaning |
|--------|---------|
| `draft` | Created, not yet sent to supplier. Lines can still be edited. |
| `open` | Confirmed with supplier. No line edits allowed. Receiving can begin. |
| `partial` | At least one unit received, not all lines fulfilled. |
| `closed` | All lines fulfilled (qty_received >= qty_ordered) AND all unit jobs closed. Auto-set by system. |
| `cancelled` | Manually cancelled. No receiving allowed. |

## PO Lines
Each PO has one or more lines. Each line = one product + quantity + price.
- `unit_price` locked at time PO moves to `open`. No editing after that.
- `qty_ordered` locked when PO status is `open` or later.
- `qty_received` increments each time a unit is received in the pipeline.

## Skip Stages Flag
Each PO has two boolean flags:
- `skip_tech` — units on this PO skip the tech inspection stage
- `skip_qa` — units on this PO skip the QA stage

Set at PO creation (draft → open transition). Cannot change after PO is open.

## PO Closure (Auto)
System auto-closes a PO when ALL of:
1. Every PO line: `qty_received >= qty_ordered`
2. Every `po_unit_job` for this PO: `status` is `passed`, `failed`, or `skipped` (i.e. closed)

This check runs after every unit job closes. Implemented in `PurchaseOrderService::checkAndClose()`.

## Reopen Rules
A closed PO can be reopened to allow receiving additional units (e.g. late delivery).

| Attempt | Who can approve |
|---------|----------------|
| 1st reopen | Manager or above |
| 2nd reopen | Manager or above |
| 3rd+ reopen | Super Admin only |

- `reopen_count` column tracks number of times PO has been reopened.
- Blocked if ANY unit from this PO is currently on the shelf (status=`shelf` in po_unit_jobs).

## Return POs (Auto-Created)
When a unit FAILS at any pipeline stage:
- System auto-creates a return PO of `type = return` linked via `parent_po_id`.
- Return PO is pre-populated with the failed unit's product and a qty of 1.
- Return PO is managed by the `po-return` module.

## Features
| # | Feature |
|---|---------|
| 1 | List POs — paginated, search by PO number/supplier, filter by status |
| 2 | View PO — detail with lines, receiving progress, unit job summary |
| 3 | Create PO — select supplier, add lines (product + qty + price) |
| 4 | Confirm PO — move draft → open, locks prices |
| 5 | Cancel PO — only when status is draft or open with no units received |
| 6 | Reopen PO — manager/super-admin only, blocked if unit on shelf |
| 7 | View PO lines — progress bar per line (qty_received / qty_ordered) |
